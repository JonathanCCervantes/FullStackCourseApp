## Concepts

### MongoDB's Data Model

| SQL term (for comparison) | MongoDB term | Meaning |
|---|---|---|
| Database | Database | A container for related collections |
| Table | **Collection** | A group of similar records |
| Row | **Document** | One record, stored as JSON-like data (technically BSON, a binary form of JSON) |
| Column | **Field** | One named piece of data within a document |

Example document in a `tasks` collection:

```json
{
  "_id": "6710f2a1c9e1b2a1d4f8e9c0",
  "title": "Buy milk",
  "description": "2% milk",
  "completed": false,
  "createdAt": "2026-06-19T10:00:00.000Z"
}
```

`_id` is a unique identifier automatically generated by MongoDB for every document — similar in purpose to the `id` field we manually generated in Module 08, but globally unique and built-in.

### Why MongoDB Suits This Course

Unlike SQL databases, MongoDB documents don't require a rigid, predefined table schema at the database level — each document's shape can vary, and the shape closely mirrors the JavaScript objects and JSON you already know from Modules 05/08. This removes one extra concept (SQL syntax) while you're still learning the rest of the stack.

## Step-by-Step Hands-On, Part 1: Raw CRUD in the Mongo Shell

This part uses `mongosh` directly — no Node, no code files yet — purely to understand MongoDB itself before wrapping it in code.

### Step 1: Open the Shell

```
mongosh
```

### Step 2: Create/Switch to a Database

```javascript
use taskflow_db
```

MongoDB creates the database the first time you actually save something into it — running `use` alone doesn't create it yet, but switches your shell's context to it.

### Step 3: Create (Insert) Documents

```javascript
db.tasks.insertOne({ title: "Buy milk", description: "2% milk", completed: false })
```

`db.tasks` refers to a collection named `tasks` — it's created automatically on first insert. You'll see a response confirming `acknowledged: true` and a new `insertedId`.

Insert a few more for practice:

```javascript
db.tasks.insertOne({ title: "Walk dog", description: "Around the block", completed: true })
db.tasks.insertOne({ title: "Read book", description: "30 minutes", completed: false })
```

### Step 4: Read (Query) Documents

```javascript
db.tasks.find()
```
Returns all documents in the collection.

```javascript
db.tasks.find({ completed: false })
```
Returns only documents matching that filter.

```javascript
db.tasks.findOne({ title: "Walk dog" })
```
Returns just the first match.

For nicer formatting in the shell, chain `.pretty()`:
```javascript
db.tasks.find().pretty()
```

### Step 5: Update Documents

```javascript
db.tasks.updateOne(
  { title: "Buy milk" },
  { $set: { completed: true } }
)
```
`$set` updates only the specified fields, leaving the rest of the document untouched. The first argument is the filter (which document to update); the second is the update operation.

### Step 6: Delete Documents

```javascript
db.tasks.deleteOne({ title: "Read book" })
```

Confirm with `db.tasks.find()` that only two tasks remain.

### Step 7: Clean Up Before Moving to Code

To avoid confusing leftover test data with what your app creates later, clear the collection:

```javascript
db.tasks.deleteMany({})
```

This deletes everything in `tasks` (filter `{}` matches all documents) but keeps the database and collection structure.

Exit the shell:
```javascript
exit
```

## Step-by-Step Hands-On, Part 2: Connecting Node to MongoDB with Mongoose

**Mongoose** is a popular npm package that lets you define structured schemas in JavaScript and interact with MongoDB using clean, promise-based JavaScript functions instead of raw shell commands.

### Step 1: Install Mongoose and dotenv

```
npm install mongoose dotenv
```

`dotenv` loads environment variables from a `.env` file into `process.env`, keeping configuration (and later, secrets) out of your code.

### Step 2: Create Your `.env` File

1. Create a file at the project root: `.env`
   (Recall from Module 06: this file is already listed in `.gitignore` and must never be committed.)
2. Paste this exact content:

```
MONGODB_URI=mongodb://localhost:27017/taskflow_db
PORT=3000
```

This is the standard local MongoDB connection address, pointed at the same `taskflow_db` database you used in the shell above.

### Step 3: Create a Database Connection File

1. Create a new folder at the project root: `config`
2. Inside it, create `config/db.js` with this exact content:

```javascript
// config/db.js - establishes the connection to MongoDB

const mongoose = require("mongoose");

async function connectDB() {
  try {
    await mongoose.connect(process.env.MONGODB_URI);
    console.log("MongoDB connected successfully.");
  } catch (error) {
    console.error("MongoDB connection failed:", error.message);
    process.exit(1); // stop the app — it can't run without a database
  }
}

module.exports = connectDB;
```

### Step 4: Wire the Connection into `server.js`

Open `server.js` and make these exact edits:

1. At the very top of the file, before any other code, add:
   ```javascript
   require("dotenv").config();
   ```
   This must run first, so every other file can read `process.env` values.

2. Just below your existing `const express = require("express");` line, add:
   ```javascript
   const connectDB = require("./config/db");
   ```

3. Just below `const PORT = 3000;`, change it to read from the environment, and call the connection function. Replace:
   ```javascript
   const PORT = 3000;
   ```
   with:
   ```javascript
   const PORT = process.env.PORT || 3000;
   connectDB();
   ```

### Step 5: Run and Verify

```
npm run dev
```

You should now see, in order:
```
MongoDB connected successfully.
TaskFlow server running at http://localhost:3000
```

If you instead see `MongoDB connection failed: ...`, check: is MongoDB actually running (revisit Module 02 Step 5's verification)? Is `.env`'s `MONGODB_URI` spelled exactly right with no extra spaces?

1. In `mongosh`, switch to a brand-new database called `practice_db`, insert two documents into a collection called `notes` with fields `text` and `pinned` (boolean), then query for only the pinned ones.
2. In `mongosh`, practice `updateOne` with the `$set` operator to change one field on one document, and confirm the change with `find()`.

## Quiz Answer Key

1. A document.
2. `db.tasks.find({ completed: false })` returns only documents matching that filter (an empty object `{}` or no argument at all returns every document in the collection).
3. It updates only the specified field(s) on the matched document(s), leaving all other fields unchanged.
4. It reads key-value pairs out of a `.env` file and loads them into `process.env`, so your code can read configuration/secrets without hard-coding them into source files.
5. Because any code that reads `process.env` values needs those values to already be loaded; if `dotenv.config()` ran later (or not at all), `process.env.MONGODB_URI` etc. would be `undefined` when first read.
6. It logs the error and calls `process.exit(1)` to stop the entire app, because TaskFlow cannot function at all without its database — failing loudly and immediately is safer than running in a broken state.
7. It contains configuration and secrets (like database connection strings, and later passwords/keys) that should never be shared publicly or stored in version control history.
